# 🚀 ClickUp + Supabase Automation Showcase\n\n**A comprehensive automation system that integrates ClickUp for task management, Supabase for data storage, and Node.js for intelligent workflow automation.**\n\n![Status](https://img.shields.io/badge/status-active-brightgreen)\n![Node.js](https://img.shields.io/badge/node.js-v20+-green)\n![License](https://img.shields.io/badge/license-MIT-blue)\n\n---\n\n## 📋 Overview\n\nThis project demonstrates a **production-ready automation system** that:\n\n✅ **Captures leads** from multiple sources  \n✅ **Stores data** securely in Supabase PostgreSQL  \n✅ **Creates ClickUp tasks** automatically  \n✅ **Sends email notifications** in real-time  \n✅ **Schedules workflows** with cron jobs  \n✅ **Logs everything** with Winston logger  \n✅ **Validates input** with Joi schema validation  \n✅ **Handles errors** gracefully with try-catch\n\n---\n\n## 🏗️ Architecture\n\n```\n┌─────────────────┐\n│   Lead Source   │\n│  (Web Form)     │\n└────────┬────────┘\n         │\n         ▼\n┌─────────────────────────┐\n│   Express Server        │\n│  POST /webhook/lead     │\n└────────┬────────────────┘\n         │\n    ┌────┴─────┬──────────┬─────────────┐\n    │           │          │             │\n    ▼           ▼          ▼             ▼\n┌────────┐ ┌────────┐ ┌────────┐ ┌──────────┐\n│Supabase│ │ClickUp │ │ Email  │ │ Logger   │\n│(Store) │ │(Tasks) │ │(Notify)│ │(Monitor) │\n└────────┘ └────────┘ └────────┘ └──────────┘\n```\n\n---\n\n## 🛠️ Tech Stack\n\n| Component | Technology |\n|-----------|------------|\n| **Backend** | Node.js + Express.js |\n| **Database** | Supabase (PostgreSQL) |\n| **Task Management** | ClickUp API |\n| **Email** | Nodemailer |\n| **Scheduling** | node-cron |\n| **Validation** | Joi |\n| **Logging** | Winston |\n| **Testing** | Jest + Supertest |\n\n---\n\n## 📦 Installation\n\n### Prerequisites\n\n- Node.js v20+\n- npm or yarn\n- Supabase account\n- ClickUp account with API access\n- Gmail account (for email notifications)\n\n### Quick Start\n\n1. **Clone the repository**\n\n```bash\ngit clone https://github.com/ummerjadoon/clickup-supabase-automation.git\ncd clickup-supabase-automation\n```\n\n2. **Install dependencies**\n\n```bash\nnpm install\n```\n\n3. **Configure environment**\n\n```bash\ncp .env.example .env\n```\n\nEdit `.env` with your credentials:\n\n```env\nCLICKUP_API_KEY=your_key_here\nCLICKUP_TEAM_ID=your_team_id\nCLICKUP_LIST_ID=your_list_id\nSUPABASE_URL=https://your-project.supabase.co\nSUPABASE_ANON_KEY=your_key_here\nSUPABASE_SERVICE_ROLE_KEY=your_key_here\nEMAIL_USER=your_email@gmail.com\nEMAIL_PASSWORD=your_app_password\nPORT=3000\nNODE_ENV=development\n```\n\n4. **Start the server**\n\n```bash\nnode server.js\n```\n\nServer will run on `http://localhost:3000`\n\n---\n\n## 📡 API Endpoints\n\n### Health Check\n\n```bash\nGET /health\n```\n\n**Response:**\n```json\n{\n  \"status\": \"healthy\",\n  \"timestamp\": \"2024-01-15T10:30:00.000Z\"\n}\n```\n\n### Submit Lead\n\n```bash\nPOST /webhook/lead\nContent-Type: application/json\n\n{\n  \"name\": \"John Doe\",\n  \"email\": \"john@example.com\",\n  \"phone\": \"1234567890\",\n  \"company\": \"Acme Corp\",\n  \"message\": \"Interested in your services\"\n}\n```\n\n**Response:**\n```json\n{\n  \"success\": true,\n  \"data\": {\n    \"leadId\": 1,\n    \"clickupTaskId\": \"abc123\",\n    \"status\": \"processed\"\n  }\n}\n```\n\n### Server Statistics\n\n```bash\nGET /stats\n```\n\n**Response:**\n```json\n{\n  \"uptime\": 3600,\n  \"timestamp\": \"2024-01-15T10:30:00.000Z\",\n  \"environment\": \"development\"\n}\n```\n\n---\n\n## 📁 Features\n\n### 1. **Lead Capture & Processing**\n- Validates incoming lead data with Joi\n- Stores leads in Supabase PostgreSQL\n- Automatically creates ClickUp tasks\n- Assigns priority based on lead source\n\n### 2. **Email Notifications**\n- Sends real-time email on lead submission\n- HTML email templates\n- Customizable subject lines\n- Error handling with retry logic\n\n### 3. **Workflow Automation**\n- Cron-based job scheduling\n- Weekly digest generation\n- Automated task assignment\n- Status tracking\n\n### 4. **Logging & Monitoring**\n- Winston logger with multiple transports\n- Separate error and combined logs\n- Timestamp and service metadata\n- Stack trace capture\n\n### 5. **Error Handling**\n- Try-catch blocks for all operations\n- Detailed error messages\n- Graceful fallbacks\n- User-friendly responses\n\n---\n\n## 🧪 Testing\n\n### Run Tests\n\n```bash\nnpm test\n```\n\n### Manual Testing with cURL\n\n```bash\n# Health check\ncurl
